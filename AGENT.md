# AGENT.md

## 项目定位
这个项目是一个学习型实验仓库，用来学习 AI 编程语法、AI agent 基础模式，以及它们在 TypeScript / JavaScript / Python 中的表达方式。

目标是：
- 能看懂
- 能运行
- 能对比
- 能逐步积累

这不是一个默认按生产系统标准扩展的项目。

## 项目调性
- 先小后大，先跑通再抽象
- 先学习价值，再工程包装
- 先直接表达，再封装复用
- 先单一能力，再复杂系统

这里更适合清晰、直接、可拆解的实现，不适合一开始就堆很多框架、层次和术语。

## 主要学习方向
围绕以下主题逐步展开：
1. LLM 基础调用
2. Prompt 组织与复用
3. Structured Output / JSON 输出
4. Tool Calling
5. Agent Loop
6. Memory / RAG
7. Workflow / Multi-agent

## 工作原则
- 新内容优先做成最小可运行示例。
- 同一个 AI 能力，优先考虑给出 TypeScript 和 Python 两种写法，便于对照学习。
- 代码应优先体现 AI 概念本身，不要被无关工程细节淹没。
- 注释重点解释“为什么这样做”，不是逐行翻译代码。
- 可以保留学习痕迹，但不要留下含糊的半成品。

## 风格约束
- 保持中文说明。
- 保持命名直观，尽量贴近 AI 概念。
- 单个文件和单个示例尽量聚焦一个主题。
- 优先可读性，不追求“高级感”。

## 判断标准
一个改动如果满足以下条件，就符合这个项目的调性：
- 能帮助理解一个明确的 AI 概念
- 能独立运行或被快速验证
- 不依赖过多背景知识
- 以后容易和其他示例做横向对比

## 架构文档规范
- 架构文档统一放在 `docs/architecture/` 下面。
- 当前默认版本目录为 `docs/architecture/v0.0.1/`。
- 某个阶段的架构说明、模块关系、设计决策，优先放到对应版本目录中。
- 架构文档需要包含对应的架构图，用来直观看清模块边界、关系和数据流。
- 架构图与文字说明放在同一个版本目录下，避免图文分散。
- 如果是同一版本内的小调整，直接更新该版本目录里的文档。
- 如果架构已经发生明显阶段性变化，创建新的版本目录，例如 `docs/architecture/v0.0.2/`。
- `docs/architecture/INDEX.md` 作为架构文档总入口，负责说明目录结构，并索引各版本文档。
- 后续新增架构文档时，需要同步更新 `docs/architecture/INDEX.md`。
